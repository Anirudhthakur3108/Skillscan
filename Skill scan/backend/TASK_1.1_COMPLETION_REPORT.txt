## Database Schema Overview

### Table 1: students (8 columns)
- id (PK), email (UNIQUE), password_hash, full_name
- user_type (MBA/BCA), profile_data (JSON)
- created_at, updated_at (UTC timestamps)

### Table 2: skills_taxonomy (6 columns)
- id (PK), name (UNIQUE), category, description
- industry_benchmark (1-10), subcategories (JSON)
- target_users (JSON: ["MBA", "BCA"])

### Table 3: student_skills (5 columns)
- id (PK), student_id (FK), skill_id (FK)
- proficiency_claimed (1-10), source (resume/manual/assessment)
- created_at (timestamp)
- UNIQUE(student_id, skill_id)

### Table 4: assessments (8 columns)
- id (PK), student_id (FK), skill_id (FK)
- assessment_type (mcq/coding/case_study)
- difficulty_level (easy/medium/hard)
- questions (JSON), status (generated/in_progress/completed)
- created_at, updated_at (UTC)

### Table 5: assessment_responses (4 columns)
- id (PK), assessment_id (FK), student_id (FK)
- responses (JSON), submitted_at (UTC timestamp)

### Table 6: skill_scores (9 columns)
- id (PK), student_id (FK), skill_id (FK), assessment_id (FK)
- score (1-10), gaps_identified (JSON), reasoning (TEXT)
- ai_confidence (0-1 decimal), scored_at (UTC timestamp)

### Table 7: learning_plans (8 columns)
- id (PK), student_id (FK), skill_id (FK)
- gap_category (VARCHAR), recommendations (JSON)
- estimated_hours (INT), priority_score (1-5)
- created_at (UTC timestamp)
- UNIQUE(student_id, skill_id, gap_category)

### Table 8: demo_accounts (7 columns)
- id (PK), username (UNIQUE), email (UNIQUE)
- password_hash, user_type (MBA/BCA)
- prefilled_data (JSON), created_at (UTC timestamp)


# ============================================================================
# USAGE EXAMPLES
# ============================================================================

### Quick Start
```bash
cd backend
pip install -r requirements.txt
python __init__.py --init          # Create tables
python __init__.py --demo          # Add demo data
python __init__.py --verify        # Verify schema
```

### Using Models in Flask Routes
```python
from database import DatabaseManager
from models import Student, SkillsTaxonomy

db = DatabaseManager.get_session()
try:
    students = db.query(Student).all()
    skills = db.query(SkillsTaxonomy).filter_by(category="Technical").all()
finally:
    db.close()
```

### Creating Records
```python
from models import Student
from database import DatabaseManager

db = DatabaseManager.get_session()

# Create new student
student = Student(
    email="user@example.com",
    password_hash="hashed_password",
    full_name="John Doe",
    user_type="MBA",
    profile_data={"phone": "1234567890"}
)
db.add(student)
db.commit()
```

### Querying with Relationships
```python
student = db.query(Student).filter_by(email="user@example.com").first()
if student:
    print(f"Skills: {len(student.student_skills)}")
    print(f"Assessments: {len(student.assessments)}")
    for plan in student.learning_plans:
        print(f"Plan: {plan.gap_category} - Priority: {plan.priority_score}")
```
